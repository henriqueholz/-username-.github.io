Segue um tutorial simples e rápido de como criar um blog e hospedar no GitHub de graça:

Segurança no NODE

Express-brute

ferramenta que limita requisições dentro de uma rota especifica. Fieto principalmente para rotas de autenticação, impedindo o usuário de ficar tentando usar usuários e senhas aleatórios até conseguir.
Incremental, ou seja, vai incrementando o tempo de bloqueio do usuário a medida que vai aumentando o número de tentativas de login fracassadas.
Instalando:

o express-brute-redis vai servir para armazenar no bando de dados NoSQL redis o número de tentativas de cada usuário.
Usado como middleware nas rotas.

helmet

midlleware do express que adicona varios tipos de configuracoes nos headers das nossas respostas para habilitar alguns tipos de protecao como por exemplo cross-site scripting e outras injeções cross-site.

Exemplos de proteção: Impede o usuário de rodar scripts em cima de nossa aplicação, impede de usar nossa aplicação em um iframe, entre outros.

express-rate-limit

parecido com o brute, porem utilizado em todas as paginas da aplicação.
Impede um usuário de fazer farias requisições simultaneas em determinada rota da API com inteções maliciosas como por exemplo derrubar a aplicação. O rate-limit limita o número de requisições possíveis.
Precisa de uma conexão mais direta com o redis.

Cors

Usado somente para front end. Bloqueia outras aplicações front end de fazer requisições AJAX à sua aplicação.

Github

Ficar atento aos avisos de segurança do github. Habilitando em settings, data services, marcar Allow GitHub to perform read-only analysis of this repository e marcar Security Alerts.
Depois na aba “Security” marcar Automated security fixes. Ali o github faz pull requests para acertar seu código.

Greenkeeper

Atualizar automaticamente o número das versões das bibliotecas usadas. Também entra como pull request no GitHub, porém é necessário testes CI para verificar se esta funcionando.
